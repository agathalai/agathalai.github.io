Integration support

This page provides a list of third-party integrations supported by Cmd, and details about adding the integration through the web interface.

NOTE: Links under the following headings lead to third-party sites.

Slack

The Slack integration allows Cmd to send messages to Slack as a trigger action.

To add a Slack integration to Cmd, you must add the Incoming WebHooks app to your Slack team. When you add the app to your Slack team, Slack will generate the URL required to complete the integration with Cmd.

For more information, see the Incoming Webhooks article in the Slack API documentation.

Jira

The Jira integration allows Cmd to create or update Jira issues as a trigger action.

To add a Jira integration to Cmd, you must have an existing Atlassian site, and an active username and password that Cmd can use to communicate with Jira.

When you enter the Project ID, be aware that the field is case sensitive. Make sure the capitalization matches the project ID prefix as it is listed in Jira. For example, you have issue ID "CMD-123", CMD would be the project ID prefix.

HipChat

The HipChat integration allows Cmd to send messages to HipChat as a trigger action.

To add a HipChat integration to Cmd, you must create a Send Message webhook through the HipChat API.

For more information, see the Send Message article from the HipChat API Reference.

PagerDuty

The PagerDuty integration allows Cmd to send notifications to your PagerDuty account as a trigger action.

To add a PagerDuty integration to Cmd, you must add a generic events API integration using API v1 to your PagerDuty account.

For more information, see the Create a Generic Events API Integration article in the PagerDuty knowledge base.

Duo Security

The Duo Security integration allows Cmd to use Duo to authenticate access or commands as a trigger action.

To add a Duo Security integration to Cmd, you must add Auth API to your Duo Applications list.

For more information, see the First Steps section of the Duo Auth API documentation.

Webhook

The Webhook integration allows you to create custom integrations for Cmd.

Sumo Logic

The Sumo Logic integration allows Cmd to send logged trigger details to Sumo Logic.

To add a Sumo Logic integration to Cmd you must add an HTTP source to your Sumo Logic account.

For more information, see the HTTP Source article in the Sumo Logic Doc Hub.

The extra parameters field allows you to include extra log details. You can enter any value in JSON format. For example:

{ "source": "Cmd", "project": "Test" }

Be aware that Cmd has pre-defined values for the following strings, and will always overwrite your parameters with the listed values:

  • "action" = trigger.Actions
  • "ip" = runner.Session.CreatedIP
  • "date" = triggerDate
  • "howconnected" = runner.Session.HowConnected
  • "projectid", _ = globalid.To("Project", runner.Session.ProjectID)
  • "risklevel" = trigger.RiskLevel
  • "servercmdid" = serverCmdID
  • "serverid", _ = globalid.To("Server", runner.Server.ID)
  • "triggerid", _ = globalid.To("Trigger", trigger.ID)
  • "type" = triggeredType

SAML

The SAML integration allows a third-party single sign-on (SSO) service to log into Cmd on your behalf.

NOTE: You can only add one SAML integration to Cmd.

To add a SAML integration to Cmd, you must provide the SSO service with the Cmd SSO URL, and provide Cmd with the SSO service's details and X.509 certificate.

The Cmd SSO URL is generated for you when you add the integration in Cmd. Consult your SSO service provider for the other required details.
