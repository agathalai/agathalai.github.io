Global admin setup

If you are a Global admin, you are responsible for setting up the system. There are two types of administrators in the system: project administrator and global admin. For more information about project administrators, see Project administrator setup.

A Global admin is a super user and is able to make changes across projects and apply system-wide defaults. As a Global admin, it is recommended that you complete the following steps when configuring the system for the first time:

  1. After you log in, it is highly recommended that you setup 2-factor authentication for yourself — 2-factor authentication.
  2. Create any other Global admin that may be required by your organization — Users.
  3. Create projects to help organize servers by development or product environment — see Projects .
  4. Create server groups to help you quickly identify servers within a project — see Server groups.
  5. Assign global default permissions for the following features:
    1. User login methods — see User access.
    2. Commands the system is permitted to execute on your behalf — see Company settings.
    3. Actions that must be performed with every trigger — see Default actions.
    4. Enable third-party integrations across projects — see Integration permissions.
  6. You can set up the remaining project features yourself, or delegate them to other users with the appropriate permissions:

    If you have more than one project, you will need to set up the following settings in each project. For more information, see Switching projects.

    1. Add users and assign each person a role in at least one project— see Users.
    2. Add servers — see Adding a new server.
    3. Add productivity and multi-factor authentication integrations — see Integrations.
    4. Setup triggers — see Adding a trigger.
    5. Setup reports — see Creating a report.
    6. Monitor alerts — see Monitoring alerts.
  7. If you will be reviewing trigger alerts on a regular basis, setup your Home page — see Customizing your Home page.
