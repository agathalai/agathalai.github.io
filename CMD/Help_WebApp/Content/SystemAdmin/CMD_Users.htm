Users

The Users page displays a list of users that have been added to the system.

At the top of the page are three charts:

  1. Active users
  2. Most active users
  3. 2-factor authentication rate

Below is the users list. The list provides an overview of all user names, credentials, and current status.

  • Click on the far right to show or hide headings that are relevant to you.
    • Select Toggle density to change the amount of spacing in each row.
  • Click a heading to sort the list by that heading.
  • Click and drag a heading to move the column left or right.
  • Click a user's Name to see the account information.
  • Use the search bar to locate specific users by entering a name or email.

Manage users

  1. In the top-right corner, click .

    The My account page is displayed.

  2. From the left menu pane, click Users.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  3. To modify a user, click at the end of the user row then select any of the available options:

    Edit

    Select this option to edit the user account.

    1. In the following Edit User dialog, make any required changes then click Save.
    2. If you are prompted, enter your password then click Verify.
    Reset password

    Select this option to send the user a password reset email.

    If you are prompted, enter your password then click Verify.

    Reset one-time codes

    Select this option to reset the user's one-time backup codes.

    NOTE: Any previously generated backup codes become invalid.

    1. When the confirmation message is displayed, click Reset one-time codes.
    2. If you are prompted, enter your password then click Verify.
    3. When the new codes are generated, print a copy of the codes and send it to the user.
    Clear 2-factor

    If a user loses their mobile device, select this option to clear the 2-factor authentication settings from their account. Once cleared, the user can access the web interface and update their account.

    1. When the confirmation message is displayed, click Clear 2-factor.
    2. If you are prompted, enter your password then click Verify.

    The user receives an email notification that the 2-factor authentication settings on their account have been cleared. They can now log into the web interface and configure 2-factor authentication with their new mobile device.

    Deactivate account

    Select this option to deactivate or activate the user. A deactivated user is still listed in the system but cannot log in until they are activated again.

    1. When the confirmation message is displayed, click Deactivate or Activate.
    2. If you are prompted, enter your password then click Verify.

    Deactivated users are labeled as Deactivated. Active users are not labeled.

    Unlock account

    This option is only displayed if the user is locked out of their account. This occurs when the user has attempted to log in with the wrong credentials too many times.

    Account locks are temporary and automatically expire after 1 hour, but you can manually unlock an account to grant immediate access.

    Select this option to restore the user's account access.

    1. When the confirmation message is displayed, click Unlock account.
    2. If you are prompted, enter your password then click Verify.
    Delete

    Select this option to deactivate and delete the user from the system.

    1. When the confirmation message is displayed, click Delete.
    2. If you are prompted, enter your password then click Verify.

Add a new user

To add an individual user, complete the following steps:

  1. In the top-right corner, click .

    The My account page is displayed.

  2. From the left menu pane, click Users.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  3. On the right side of the page, click Add user
  4. In the following dialog, add the new user information:
    • Name — enter the name of the user.
    • Email address — enter the user's email.
    • User name — enter the user's Linux system username.
    • Department — enter any LDAP or Active Directory user group that is used to help define your organization.
    • Job category — assign a job category to the user.

      Click the field to select a category from the available options, or start typing to automatically filter the options.

      Job categories are defined as part of the User access settings. For more information, see User access.

  5. If the user needs to be a superuser, click to enable Global admin access.

    As a Global admin, the user would have access to all projects, servers, and features in the system. You will not be able to limit the user's access to a specific project or server group. For more information about what a Global admin can do, see Global admin setup.

  6. In the Project roles area, select any of the listed projects on the left, then assign a role to the user on the right.

    This setting does not apply to Global admin users. The user can be assigned one of the following roles in each project:

    • None — the user does not have access to the project.
    • Administrator — the user is able to add, edit, and remove projects, servers and users from the project. For more information about what an Administrator can do, see Global admin setup.
    • Manager — the user is able to add servers and manage triggers and reports. For more information about what a manager can do, see Manager setup.
      • Click the Server group restriction field to select the server groups that the user may access. Leave empty to grant access to all servers within the project.
    • Viewer — the user is able to monitor alerts and create reports. For more information about what a viewer can do, see Viewer setup.
      • Click the Server group restriction field to select the server groups that the user may access. Leave empty to grant access to all servers within the project.
    • Server operator — the user can only access the web interface to configure 2-factor authentication for normal server operations. The user does not have access to any other web interface feature. For more information, see Server operators.
      • Click the Server group restriction field to select the server groups that the user may access. Leave empty to grant access to all servers within the project.
  7. Click Save.
  8. If you are prompted, enter your password then click Verify.

The system sends the new user an email notifying them that they now have access to the Cmd web interface. The email notification they receive contains a link to set a new password. After their password has been set, the new user will be prompted to login with their email and new password. For more information, see Logging in for the first time.

Add multiple users

If you need to add a large number of users, you have the option of adding all the users simultaneously through a CSV (comma separated value) file. You can use the following file as a template: multi-user-template.csv

The CSV file must list users in this format: FirstName LastName,Email

For example, John Doe,johndoe@gmail.com

To bulk upload users to the system, complete the following steps:

  1. Prepare a CSV file with all the user names and emails.
  2. In the Cmd web interface, access the Users page.
  3. On the right side of the page, click Add multiple users.
  4. In the following dialog, locate and select the prepared CSV file.
  5. Click Upload CSV.
  6. If you are prompted, enter your password then click Verify.

All the users in the CSV file are added to the system. Next, edit each user and assign them a role in the system.
