Integration permissions

The Integration permissions page allows you to enable or disable global system access to third-party integrations.

Integrations are project specific, so you would need to add an integration to each project before you can create triggers that use the integrations. For more information, see Integrations.

If you are trying to disable an integration, you must remove the integration action from all triggers before you are permitted to disable the integration.

To enable or disable an integration, complete the following steps:

  1. In the top-right corner, click .

    The My account page is displayed.

  2. From the left menu pane, click Integration permissions.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  3. On the Integration permissions page, click the toggle for the integration type.
  4. Click Save.
  5. If you are prompted, enter your password then click Verify.

If you are unable to disable an integration because there are still outstanding triggers using the integration, check the Triggers page and the Default actions page, then try again.
