User access

The User access page defines the general user permissions for accessing the Cmd web interface.

Enable or disable any of the following settings, then click Save.

Option Description

Allow change email

Enable this setting to allow users to change the email used to access the web interface.

Password change settings

  • Allow forgot password — enable this setting to allow the system to assist users if they forget their password.
  • Enforce password change — enable this setting to require users to change their passwords regularly.

    When enabled, you can set the maximum number of days before users are prompted to change their password.

  • Enforce unique password on change — enable this setting to require users to set a unique password each time it is changed.

Allow Google login

Enable this setting to allow users to access the Cmd web interface with their Google credentials.

To log in with Google credentials, users must configure their Cmd account to use their Google account email.

  • Enforce Google login — enable this setting if users must use their Google account as their login credential. Any other email credential would not be permitted.
Enforce 2-factor authentication

Enable this setting if users must be verified by 2-factor authentication to log into the web interface.

To configure 2-factor authentication for yourself, see 2-factor authentication.

To send users a request to configure 2-factor authentication, select the Clear 2-factor option from the Users page. For more information, see Users.
Session timeout

Enable this setting to automatically log users out of the web interface after they have been idle for too long.

When enabled, you can set the maximum amount of time (in minutes) that the user can be inactive before they are logged out.
Restrictions
  • Restrict emails to specific domains — enable this setting if all user emails must be from a specific domain.

    When enabled, enter the required domains then press Enter.

  • Restrict logins to specific IP — enable this setting if users can only access the web interface from specific IP addresses.

    When enabled, enter the required IP addresses then press Enter. Wildcard * is accepted.

  • Restrict logins to specific countries — enable this setting if users are only expected to login from a specific country.

    When enabled, enter the country name then press Enter.
Allow export data Enable this setting if users are allowed to export data from the web interface.
Job categories

Enter any job category that is needed to define your organization.

Type the job category then press Enter to add it to the list. Add as many as needed.

The listed job categories can be assigned to users in the system.
SOS ping post URL Enter the URL of your preferred system health monitoring service.
