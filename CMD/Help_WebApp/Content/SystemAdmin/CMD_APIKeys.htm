Project API keys

Project API keys are used to validate the identity of servers and confirm that they are supposed to have permission to communicate with Cmd. API key are required to add servers to the system. For more information about adding servers to the system, see Adding a new server.

Listed on the Project API keys page are all the keys that have been generated. The Active tab lists any active keys, while the Canceled tab lists the keys that have been canceled or expired.

Both lists provide an overview of the API key details.

  • Click on the far right to show or hide headings that are relevant to you.
    • Select Toggle density to change the amount of spacing in each row.
  • Click an API key to automatically copy the key to your local clipboard.

Cancel

To help maintain the security of your system, it is recommended that you cancel any active keys that are not being used.

NOTE: Once a key has been canceled, it cannot be reactivated.

  1. In the top-right corner, click .

    The My account page is displayed.

  2. From the left menu pane, click Project API keys.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  3. Locate the key you want to cancel.
  4. On the right-end of the row, select > Cancel.
  5. In the confirmation dialog, click Cancel API key.
  6. If you are prompted, enter your password then click Verify.

The API key is moved to the Canceled tab.

Add

To avoid having too many active keys, it is highly recommended that you only add a new API key when you plan to add servers to the system, and only configure the keys to support the expected number of servers.

  1. In the top-right corner, click .

    The My account page is displayed.

  2. From the left menu pane, click Project API keys.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  3. On the Project API keys page, click Add API key.

    The Add API key dialog is displayed.

  4. In the Max number of installs field, enter the maximum number of servers you plan to add to the system.

    The key is automatically retired when the max number of install is reached or the Hours time limit is reached — whichever comes first.

  5. In the Hours field, enter the maximum number of hours that the key is valid for.
  6. In the IP CIDR masks field, you can choose to limit the IP addresses the key can be used on.

    Type the IP addresses then press Enter to add them to the list.

  7. Click Add API key.
