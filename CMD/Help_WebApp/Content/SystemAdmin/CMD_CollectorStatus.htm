Collector status

The Collector status page provides the health status of the Cmd collector for your system. The collector is a Cmd application that runs on a dedicated on-premise server or in a cloud environment, and is used to compile all the data captured by the servers in your network.

At the top of the page is the collector health status. It identifies the current volume of commands going through your system.

  1. IP address of the collector server.
  2. Command volume — the volume of commands executed in the last 15 minutes.
  3. Queue volume — amount of commands queued up for triggers
  4. The average number of commands per second according to the command volume.
  5. Health status — the collector can be Healthy or Unhealthy.
  6. CPU usage
  7. Memory usage
  8. Network.
