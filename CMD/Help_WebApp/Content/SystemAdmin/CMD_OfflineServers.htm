Offline servers

The Offline servers page allows you to create schedules for planned server outages. Servers listed on this page are only expected to be offline for a few hours while you perform operating system updates and any required hardware changes.

When you schedule a server to go offline, you are actually scheduling the system to shut down the Cmd client application. While the client application is offline, Cmd will not be able to communicate with the server — no data is collected and triggers are disabled. The actual server can remain online for you to perform maintenance.

Review schedules

  1. In the top-right corner, click .

    The My account page is displayed.

  2. In the left menu pane, click Offline servers.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

    The Offline servers page is displayed. The first tab you see is the All tab that provides a historical list of schedules that have been created.

  3. Click on the far right to show or hide headings that are relevant to you.
    • Select Toggle density to change the amount of spacing in each row.
  4. Click the other tabs to filter the displayed schedules by their status:
    • Active — the schedule is active and the servers are currently offline or are queued to go offline.
    • Canceled — the schedule was canceled before or while the server was offline.
    • Done — the schedule was completed.

Add

  1. In the top-right corner, click .

    The My account page is displayed.

  2. In the left menu pane, click Offline servers.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  3. In the top-right corner of the Offline servers page, click Add offline server.
  4. In the following dialog, select the server or server group that the schedule is for.

    Click the field to select a server or group from the available options, or start typing to automatically filter the options.

    Tip: You must type the server ID to find a specific server in the list. Individual servers are listed in this format: server ID (server name), for example SVR-12AB (www). The server ID can be found on the Sources page under the Server tab.

  5. Click Select date then select the schedule date in the calendar.

    By default, the current time is automatically inserted as the schedule time.

  6. At the top of the calendar, click the field to manually change the date or time.

    The time must be entered in this format: HH:mm:ss. For example, 23:45:00 would be 11:45pm.

    NOTE: The server will go offline within 5 minutes of the set time and date.

  7. In the Duration field, enter the expected number of hours that the server would be offline.
  8. In the Reason field, explain why the server will be offline.
  9. Click Add offline server.
  10. If you are prompted, enter your password then click Verify.

The new offline server schedule is added to the Active tab.

All project administrators will receive an email reminder 1 hour before the server is expected to go offline, and again 1 hour before the server returns online.

Cancel

You can cancel a schedule before or while the server is offline. If the server is already offline, you can only cancel the schedule if the actual server has power and is connected to the network.

  1. In the top-right corner, click .

    The My account page is displayed.

  2. In the left menu pane, click Offline servers.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  3. Locate the schedule that you want to cancel.
  4. In the right-end of the row, select > Cancel.
  5. In the following dialog, provide a reason.
  6. Click Save.
  7. If you are prompted, enter your password then click Verify.

The schedule is moved to the Canceled tab.

If the server was offline, wait a few minutes while communication with the server is restored.
