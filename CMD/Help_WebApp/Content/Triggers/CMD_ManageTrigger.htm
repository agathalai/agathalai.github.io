Managing triggers

Cmd provides you with a number of ways to modify and manage triggers.

The main Triggers page provides you with the option to edit, deactivate, archive or copy a trigger.

While the left menu pane gives you access to the following:

If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  • Triggers — click this option to return to the main Triggers page.

    Click to add a new trigger. For more information, see Adding a trigger.

  • Archived — click this option to see a list of triggers that are no longer used.
  • Macros — click this option to see pre-configured trigger conditions that have been saved and can be used with other triggers. For more information, see Macros.
  • Trigger groups — lists all the trigger groups that have been added to the project.

    Click a trigger group name to see the triggers in that group. For more information about trigger groups, see Trigger groups.

Edit

Tip: You can also edit archived triggers.

  1. From the Triggers page, open the Trigger details page:
    • Click the trigger name.
    • At the right-end of the trigger row, select > Edit.
    • Click to expand the trigger row, then click Edit trigger in the Conditions area.
  2. On the Trigger details page, make any required changes.

    For more information about the available options, see Adding a trigger.

  3. When you are ready, click Save at the bottom of the page.

Deactivate/activate

You can deactivate a trigger to pause the trigger from tracking trigger conditions or generating any alerts. While it is deactivated, you can still edit and modify the trigger as needed. When you are ready, you can activate the trigger again at any time.

  1. On the Triggers page, locate the trigger that you want to modify.
  2. At the right-end of the row, select > Deactivate.
  3. In the following confirmation dialog, click Deactivate or Activate.
  4. If you are prompted, enter your password then click Verify.

If you deactivated a trigger, the Deactivated label is added to the trigger.

If you reactivated a trigger, the Deactivated label is removed.

Archive

Triggers cannot be deleted, only archived. When you archive a trigger, you are removing it from active service and moving the trigger to the Archived page.

  1. On the Triggers page, locate the trigger that you want to archive.
  2. At the right-end of the row, select > Archive.
  3. In the following confirmation dialog, click Archive.
  4. If you are prompted, enter your password then click Verify.

The trigger is removed from the Triggers page, and added to the Archived page.

Restore

You can choose to restore an archived trigger to return it to active service.

  1. On the Triggers page, click Archived from the left menu pane.

    If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  2. On the right-end of the row, select > Restore.
  3. In the following dialog, click Unarchive trigger.
  4. If you are prompted, enter your password then click Verify.

Subscribe/unsubscribe

You can subscribe to a trigger to receive email notifications whenever the trigger is activated. This is separate from the Send email trigger action.

To avoid receiving too many subscription notifications, you can configure the system to only send summarized emails at set intervals. For more information, see Summarize email actions.

You can also choose to unsubscribe at any time.

  1. On the Triggers page, locate the trigger that you want.
  2. At the right-end of the row, select > Subscribe or Unsubscribe.

Next time the trigger is activated, you will receive an email notifying you that a trigger violation occurred.

If you've unsubscribed, you should stop receiving email notifications immediately.

Duplicate

NOTE: Not available for Global triggers.

  1. On the Triggers page, locate the trigger that you want to duplicate.
  2. At the right-end of the row, select > Copy trigger.
  3. In the following dialog box, select the current project.
  4. Click Copy trigger.

A copy of the trigger is added to the Triggers page. The date and time it was copied is appended to the trigger name.

Copy trigger to a different project

NOTE: Not available for Global triggers.

  1. On the Triggers page, locate the trigger that you want to copy.
  2. At the right-end of the row, select > Copy trigger.
  3. In the following dialog box, select the project you want to copy the trigger to.
  4. Click Copy trigger.

A copy of the trigger is added to the selected project's Triggers page. The date and time it was copied is appended to the trigger name.
