Adding a new server

When you add a server, you are installing the Cmd client application on to your server. Once installed, data can be collected and triggers can be configured for the server.

Before you can add a new server to the system, you need to make sure you are in the correct project and that there is an active API key available.

  1. Confirm that you are adding a server to the correct project.

    The top-right corner of the page displays the project that you are currently accessing.

    If you need to add the server to a different project, click the name of the current project then select a different project from the drop down list.

  2. Access the Project API keys page to ensure an active API key is available.

    In the top-right corner of the page, click then click Project API keys from the left menu pane.

    The Project API keys page is displayed and should list one or more active keys. If not, complete the following steps to add an API key:

    1. On the Project API keys page, click Add API key.

      The Add API key dialog is displayed.

    2. In the Max number of installs field, enter the maximum number of servers you plan to add to the system.

      The key is automatically retired when the max number of install is reached or the Hours time limit is reached — whichever comes first.

    3. In the Hours field, enter the maximum number of hours that the key is valid for.
    4. In the IP CIDR masks field, you can choose to limit the IP addresses the key can be used on.

      Type the IP addresses then press Enter to add them to the list.

    5. Click Create.
  3. Navigate to the Sources page then click Add server.
  4. On the following page, enter a server name.

    Be aware that the server name must be alphanumeric. Spaces and special characters are not supported.

  5. Select the server's operating system from the drop down list.
  6. Select an active API key from the drop down list.
  7. Assign the server to a group.

    Click the field to select a category from the available options, or start typing to automatically filter the options.

    Server groups are defined on the Company settings page. For more information, see Server groups.

  8. At the bottom of the page, click Copy to clipboard.

    This copies the generated Cmd installation command to your local clipboard.

  9. Open a command line terminal on the server you are adding to Cmd, then paste the installation command and press Enter.

When you see the success message, the server has been added to the system. You can now begin normal operations by adding users and triggers.

If you see an error message, check the following:

  • The API key is active.

    If the API key expired during the installation process, add a new API key and try again.

  • The installation command was pasted correctly.

    Each command is specific to a server, so check that the command was pasted to the correct server.

    Also, check that the pasted command did not include any extra characters or spaces. This issue typically occurs when you choose to manually copy and paste the command rather than use the Copy to clipboard function.
