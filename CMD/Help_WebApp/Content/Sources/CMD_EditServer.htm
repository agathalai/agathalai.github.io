Editing servers

You can edit a server to change its name or server group. In the same dialog, you can choose to enable or disable system permissions at the server level. These settings would override the configured global system settings and project settings.

To modify the global system settings, see Company settings.

To modify the project settings, see Set project permissions.

  1. On the Sources page, identify the servers you want to edit.
    • If you are only editing one server, select > Edit
    • If you want to edit more than one server at the same time, select the check box on the left end of each server row.

      Then click Edit selected servers in the top-right corner of the list.

  2. In the following dialog, update the server's name or group as needed.

    Be aware that if you are editing multiple servers simultaneously, you will not be able to edit the servers to use the same name.

  3. Enable or disable any of the listed permissions.

    By default, the project settings are displayed. Any changes made here would only affect the selected servers.

    • Capture output — enable to allow the Cmd application to record the results of entered commands.
      • Errors only — if a command results in an error, only capture the error message.
    • Get file diff — enable this setting to capture the differences in files when they are changed.
    • Require two factor — enable this setting to ensure users pass dual authentication before they are allowed to access the server.
    • Require Jira ticket — enable this setting to ensure users can provide a Jira ticket to capture the server access. Once enabled, select the Jira project that would capture the server access.
  4. Click Save.
