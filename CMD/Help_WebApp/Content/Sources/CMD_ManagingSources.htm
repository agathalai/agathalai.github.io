Monitoring sources

The Sources page gives you a timeline of activity across all servers in the project. You have the option of monitoring servers, sessions, or commands as they occur over time, including live activity.

To see servers for a different project, see Switching projects.

You can use any of the following tools to help you monitor the sources for events of interest.

  • Monitoring sources
    • A — Filters
    • B — Search
    • C — Timeline
    • D — Sources list
    • E — Commands view

A — Filters

From the left menu pane, click any of the listed filters. If you do not see the left menu pane, swipe to the left browser edge to show the menu.

  • Click a server group name to open a new tab to display the server group details.

    Server groups are groups that servers are assigned to. You can assign servers to a group when you edit server details. For more information, see Editing servers.

    The groups are defined on the Commands page. For more information, see Server groups.

  • Click a filter name to run a pre-configured search of the displayed data.

    For more information, see Source filters.

B — Search

At the top of the Sources page is a search bar to help you filter through all the available data. Each time you run a search, it opens a new tab on the page.

You can enter your search query manually or click to use the builder.

Enter manually

  • Click the bar to select the starting query. Use the auto-complete prompts to further narrow your query.

    For more information about the available CQL values, see CQL values.

  • Click to use the search query as any of the following:
    • Sources — select this option to add the search query as a filter for the Sources page. For more information, see Source filters.
    • Insights — select this option to create a new report from the search query. For more information, see Creating a report.
    • Triggers — select this option to use the search query as the base of a new trigger. For more information, see Adding a trigger.
Use the builder

  • On the left, click the heading titles to show the condition properties that are available.

    For more information about the listed properties, see CQL values.

  • Click All to select if All or Any of the tree conditions need to be true to activate the trigger actions.
  • Add a property to the tree then assign the required values. You can add a property in any of the following ways:
    • Click a property from the provided list.
    • Drag and drop any property from the list to the tree.
    • Click Add property.
    • Hover over any property in the tree then click .
  • Click Add group to create a subgroup of condition properties.
  • From the left end of each property branch, drag and drop the property to change it's position in the tree.
  • To delete a property from the tree, click .

You can do any of the following while performing a search:

  • Click to use the search query as any of the following:
    • Sources — select this option to add the search query as a filter for the Sources page. For more information, see Source filters.
    • Insights — select this option to create a new report from the search query. For more information, see Creating a report.
    • Triggers — select this option to use the search query as the base of a new trigger. For more information, see Adding a trigger.
  • Click to see your previous search queries. You can click any of the listed options to perform that search again.
  • Click to clear the search.

C — Timeline

Use the timeline to narrow the source activity to a specific time period. You can change the time range by doing any of the following:

  • Click the date on the top-right corner to select the date range.

    • The first date you click is always the start date. The second date you click is always the end date.
    • Double-click the same day to limit the time range to 24 hours.
    • Click any of the preset time ranges on the bottom of the calendar.
    • Click the text field at the top to manually enter a specific date and time.

      The field on the left is the start date, and the field on the right is the end date.

  • Click and drag across the timeline to zoom in to the highlighted time range.

  • While hovering over the timeline, click or to zoom in and out of the timeline.
  • The results are all grouped until you zoom in within 2 days on the timeline, then you can click Single to see individual session lengths.

  • You can pan the timeline by horizontal scrolling.

D — Sources list

At the bottom half of the Sources page is the sources list. The sources are sorted into Servers and Sessions tabs.

  • Click on the far right to show or hide headings that are relevant to you.
    • Select Toggle density to change the amount of spacing in each row.
  • Click a heading to sort the list by that heading.
  • Click and drag a heading to move the column left or right.
  • Hover over the linked text to see a summary of the server, session, or user status and the number of violations associated with it.

    • If the server summary display is red, there are violations related to the server. If it is green, there are no unresolved violations.
    • If the session summary display is red, there was a failed 2-factor authentication attempt. If it is green, 2-factor authentication was successful. If it is gray, 2-factor authentication was not setup for the user.
    • In the summary display, click any of the linked text to see more details.
    • Click Run in Sources to see the command in context in the Commands tab.
    • Click Run in Insights to create a new report about the summarized item.
  • Click on the top-right corner of the table to export a copy of the listed items in .CSV or JSON format.
    1. In the following dialog, select the preferred export format from the drop down list.
    2. Enter the number of rows that you want to export.
    3. Select the headings that you want to include in the export.
    4. Click Export.
    5. In the following dialog, right-click the bar and save the linked target file as instructed.
    6. When the file finishes downloading, click OK to close the dialog.

E — Commands view

To see commands executed within the set timeline, select the Commands tab.

To see live session activity, select the Live tab. Each live session is displayed in it's own terminal.

  • >_ command triggered a violation.
  • Click [link] to see the command output or child commands.
A. Search

Use the search bar to find specific command text.

B. Display mode

Click to display the commands in List mode.

Click to display the commands in Hierarchy mode.

C. User only

Toggle this option to only show all commands or only commands executed by a user.

D. Details

Toggle this option to show or hide the Details pane.

When you select any command in the terminal, the Details pane displays the information about the source of the command and any affected files.

In the Details pane, hover or click any linked text to see more details.

E. Scrub data

Scrubbing data redacts the command or output for all users.

NOTE: When you scrub one command, all instances of the same command is scrubbed as well.

  1. Click to manually scrub sensitive information.

    The icon is displayed in the Details pane or in the top-right corner of the command output display.

  2. When the field becomes editable, press Backspace to scrub some or all of the displayed information.
  3. Click to save your changes.
  4. In the confirmation dialog, click Scrub.
F. Customize

Click to customize the terminal display. For more information, see Configuring your terminal display.

G. Maximize

Click to maximize the terminal display.

While monitoring live commands:

  1. Displays the session ID, username, and IP address. If any violations occur during the live session, the alert risk level icon is also displayed here.
  2. Displays a running command count beside the session duration.

Click Stop session to end the live session — the session would be terminated and the end user would be logged out of the server.
